	<h1 id="firstHeading" class="firstHeading" lang="en">Semantic Interpretation for Speech Recognition</h1>

<p><b>Semantic Interpretation for Speech Recognition</b> (SISR) defines the syntax and semantics of annotations to grammar rules in the <a href="/wiki/Speech_Recognition_Grammar_Specification" title="Speech Recognition Grammar Specification">Speech Recognition Grammar Specification</a> (SRGS).  Since 5 April 2007, it is a <a href="/wiki/W3C" class="mw-redirect" title="W3C">World Wide Web Consortium</a> recommendation.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup>
</p><p>By building upon SRGS grammars, it allows <a href="/wiki/Voice_browser" title="Voice browser">voice browsers</a> via <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> to semantically interpret complex grammars and provide the information back to the application.  For example, it allows utterances like "I would like a Coca-cola and three large pizzas with <a href="/wiki/Pepperoni" title="Pepperoni">pepperoni</a> and mushrooms." to be interpreted into an object that can be understood by an application.  For example, the utterance could produce the following object named <tt>order</tt>:
</p>
<div class="mw-highlight mw-highlight-lang-javascript mw-content-ltr" dir="ltr"><pre><span></span> <span class="p">{</span>
   <span class="nx">drink</span><span class="o">:</span> <span class="p">{</span>
     <span class="nx">liquid</span><span class="o">:</span><span class="s2">&quot;coke&quot;</span><span class="p">,</span>
     <span class="nx">drinksize</span><span class="o">:</span><span class="s2">&quot;medium&quot;</span>
