	<h1 id="firstHeading" class="firstHeading" lang="en">Network operating system</h1>
	
	<div id="bodyContent" class="mw-body-content">
		<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>
		<div id="contentSub"></div>
		
		
		<div id="jump-to-nav"></div>
		<a class="mw-jump-link" href="#mw-head">Jump to navigation</a>
		<a class="mw-jump-link" href="#p-search">Jump to search</a>
		<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>A <b>network operating system</b> is a specialized <a href="/wiki/Operating_system" title="Operating system">operating system</a> for a network device such as a <a href="/wiki/Router_(computing)" title="Router (computing)">router</a>, <a href="/wiki/Network_switch" title="Network switch">switch</a> or <a href="/wiki/Firewall_(computing)" title="Firewall (computing)">firewall</a>.
</p><p>Historically operating systems with networking capabilities were described as network operating system, because they allowed personal computers (PCs) to participate in <a href="/wiki/Computer_networks" class="mw-redirect" title="Computer networks">computer networks</a> and <a href="/wiki/Shared_file_access" class="mw-redirect" title="Shared file access">shared file and printer access</a> within a <a href="/wiki/Local_area_network" title="Local area network">local area network</a> (LAN). This description of operating systems is now largely historical, as common operating systems include a <a href="/wiki/Network_stack" class="mw-redirect" title="Network stack">network stack</a> to support a <a href="/wiki/Client%E2%80%93server_model" title="Client–server model">client–server model</a>.
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Network_operating_system&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Early microcomputer operating systems such as <a href="/wiki/CP/M" title="CP/M">CP/M</a>, <a href="/wiki/DOS" title="DOS">DOS</a> and <a href="/wiki/Classic_Mac_OS" title="Classic Mac OS">classic Mac OS</a> were designed for one user on one computer.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (March 2019)">citation needed</span></a></i>&#93;</sup> <a href="/wiki/Packet_switching" title="Packet switching">Packet switching</a> networks were developed to share hardware resources, such as a <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe computer</a>, a <a href="/wiki/Printer_(computing)" title="Printer (computing)">printer</a> or a large and expensive <a href="/wiki/Hard_disk" class="mw-redirect" title="Hard disk">hard disk</a>.<sup id="cite_ref-Understanding_Operating_Systems_1-0" class="reference"><a href="#cite_note-Understanding_Operating_Systems-1">&#91;1&#93;</a></sup> As local area network technology became available, two general approaches to handle sharing of resources on networks arose.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (March 2019)">citation needed</span></a></i>&#93;</sup>
</p><p>Historically a network operating system was an <a href="/wiki/Operating_system" title="Operating system">operating system</a> for a computer which implemented network capabilities. Operating systems with a <a href="/wiki/Network_stack" class="mw-redirect" title="Network stack">network stack</a> allowed <a href="/wiki/Personal_computers" class="mw-redirect" title="Personal computers">personal computers</a> to participate in a <a href="/wiki/Client%E2%80%93server_model" title="Client–server model">client-server architecture</a> in which a <a href="/wiki/Server_(computing)" title="Server (computing)">server</a> enables multiple clients to share resources, such as <a href="/wiki/Printer_(computing)" title="Printer (computing)">printers</a>.<sup id="cite_ref-autogenerated2_2-0" class="reference"><a href="#cite_note-autogenerated2-2">&#91;2&#93;</a></sup><sup id="cite_ref-autogenerated3_3-0" class="reference"><a href="#cite_note-autogenerated3-3">&#91;3&#93;</a></sup><sup id="cite_ref-:0_4-0" class="reference"><a href="#cite_note-:0-4">&#91;4&#93;</a></sup> Early examples of client-server operating systems that were shipped with fully integrated network capabilities are <a href="/wiki/Novell_NetWare" class="mw-redirect" title="Novell NetWare">Novell NetWare</a> using the <a href="/wiki/Internetwork_Packet_Exchange" title="Internetwork Packet Exchange">Internetwork Packet Exchange</a> (IPX) network protocol, <a href="/wiki/Windows_Server_2003" title="Windows Server 2003">Windows Server 2003</a>, and <a href="/wiki/Banyan_VINES" title="Banyan VINES">Banyan VINES</a> which used a variant of the <a href="/wiki/Xerox_Network_Systems" title="Xerox Network Systems">Xerox Network Systems</a> (XNS) protocols.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (March 2019)">citation needed</span></a></i>&#93;</sup>
</p><p><a href="/wiki/Peer-to-peer" title="Peer-to-peer">Peer-to-peer</a> network operating systems were also developed, which used networking capabilities to share resources and files located on personal computers. This system is not based with having a file server or centralized management source. A peer-to-peer network sets all connected computers equal; they all share the same abilities to use resources available on the network.<sup id="cite_ref-autogenerated3_3-1" class="reference"><a href="#cite_note-autogenerated3-3">&#91;3&#93;</a></sup> Examples of early peer-to-peer operating systems with networking capabilities include <a href="/wiki/AppleShare" title="AppleShare">AppleShare</a> used for networking connecting Apple products, <a href="/wiki/LANtastic" title="LANtastic">LANtastic</a> supporting DOS, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> and <a href="/wiki/OS/2" title="OS/2">OS/2</a> computers, as well as <a href="/wiki/Windows_for_Workgroups" class="mw-redirect" title="Windows for Workgroups">Windows for Workgroups</a> used for networking peer-to-peer Windows computers.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (March 2019)">citation needed</span></a></i>&#93;</sup>
</p><p>Today, <a href="/wiki/Distributed_computing" title="Distributed computing">distributed computing</a> and <a href="/wiki/Groupware" class="mw-redirect" title="Groupware">groupware</a> applications have become the norm. Computer operating systems include a networking stack as a matter of course.<sup id="cite_ref-Understanding_Operating_Systems_1-1" class="reference"><a href="#cite_note-Understanding_Operating_Systems-1">&#91;1&#93;</a></sup> During the 1980s the need to integrate dissimilar computers with network capabilities grew and the number of networked devices grew rapidly. Partly because it allowed for multi-vendor <a href="/wiki/Interoperability" title="Interoperability">interoperability</a> the <a href="/wiki/TCP/IP" class="mw-redirect" title="TCP/IP">TCP/IP</a> protocol suite became almost universally adopted in network architectures. Therefore, computer operating systems and the <a href="/wiki/Firmware" title="Firmware">firmware</a> of network devices needed to reliably support the TCP/IP protocols.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Network_device_operating_systems">Network device operating systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Network_operating_system&amp;action=edit&amp;section=2" title="Edit section: Network device operating systems">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Network operating systems can be embedded in a <a href="/wiki/Router_(computing)" title="Router (computing)">router</a> or <a href="/wiki/Firewall_(computing)" title="Firewall (computing)">hardware firewall</a> that operates the functions in the <a href="/wiki/Network_layer" title="Network layer">network layer</a> (<a href="/wiki/OSI_model" title="OSI model">layer 3</a>).<sup id="cite_ref-expert_6-0" class="reference"><a href="#cite_note-expert-6">&#91;6&#93;</a></sup> Notable network operating systems include:
</p>
<h3><span class="mw-headline" id="Proprietary_network_operating_systems">Proprietary network operating systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Network_operating_system&amp;action=edit&amp;section=3" title="Edit section: Proprietary network operating systems">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li><a href="/wiki/Cisco_IOS" title="Cisco IOS">Cisco IOS</a>, a family of network operating systems used on most <a href="/wiki/Cisco_Systems" title="Cisco Systems">Cisco Systems</a> <a href="/wiki/Router_(computing)" title="Router (computing)">routers</a> and current Cisco <a href="/wiki/Network_switch" title="Network switch">network switches</a>. Earlier switches ran the Catalyst Operating System <a href="/wiki/CatOS" class="mw-redirect" title="CatOS">CatOS</a></li>
<li><a href="/wiki/MikroTik#RouterOS" title="MikroTik">RouterOS</a> by <a href="/wiki/MikroTik" title="MikroTik">MikroTik</a>.</li>
<li><a href="/wiki/ZyNOS" title="ZyNOS">ZyNOS</a>, used in network devices made by <a href="/wiki/ZyXEL" class="mw-redirect" title="ZyXEL">ZyXEL</a>.</li></ul>
<h3><span id="NetBSD.2C_FreeBSD.2C_or_Linux_based_operating_systems"></span><span class="mw-headline" id="NetBSD,_FreeBSD,_or_Linux_based_operating_systems">NetBSD, FreeBSD, or Linux based operating systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Network_operating_system&amp;action=edit&amp;section=4" title="Edit section: NetBSD, FreeBSD, or Linux based operating systems">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<ul><li><a href="/wiki/DD-WRT" title="DD-WRT">DD-WRT</a>, Linux kernel based DD-WRT is Linux-based firmware for wireless routers and access points as well as low-cost networking device platforms such as the <a href="/wiki/Linksys" title="Linksys">Linksys</a> <a href="/wiki/Linksys_WRT54G_series" title="Linksys WRT54G series">WRT54G</a></li>
<li><a href="/wiki/Dell_Networking_Operating_System" title="Dell Networking Operating System">Dell Networking Operating System</a>, DNOS9 is <a href="/wiki/NetBSD" title="NetBSD">NetBSD</a> based, while OS10 uses the <a href="/wiki/Linux_kernel" title="Linux kernel">Linux kernel</a>.</li>
<li><a href="/wiki/Arista_Networks#Extensible_Operating_System" title="Arista Networks">Extensible Operating System</a> runs on switches from <a href="/wiki/Arista_Networks" title="Arista Networks">Arista</a> and uses an unmodified Linux kernel</li>
<li><a href="/wiki/ExtremeXOS" title="ExtremeXOS">ExtremeXOS</a> (EXOS), used in network devices made by <a href="/wiki/Extreme_Networks" title="Extreme Networks">Extreme Networks</a></li>
<li><a href="/wiki/FTOS" title="FTOS">FTOS</a> or Force10 Operating System, is the <a href="/wiki/Firmware" title="Firmware">firmware</a> family used on <a href="/wiki/Force10" title="Force10">Force10</a> Ethernet switches</li>
<li><a href="/wiki/OpenWrt" title="OpenWrt">OpenWrt</a> used to route <a href="/wiki/Internet_Protocol" title="Internet Protocol">IP packets</a> on <a href="/wiki/Embedded_devices" class="mw-redirect" title="Embedded devices">embedded devices</a></li>
<li><a href="/wiki/PfSense" title="PfSense">pfSense</a>, a fork of <a href="/wiki/M0n0wall" title="M0n0wall">M0n0wall</a>, uses <a href="/wiki/PF_(firewall)" title="PF (firewall)">PF</a></li>
<li><a href="/wiki/Cumulus_Networks#Products" title="Cumulus Networks">Cumulus Linux</a> distribution, which uses the full TCP/IP stack of <a href="/wiki/Linux" title="Linux">Linux</a>.</li>
<li><a href="/wiki/VyOS" title="VyOS">VyOS</a>, an open source fork of the Vyatta routing package</li></ul>
