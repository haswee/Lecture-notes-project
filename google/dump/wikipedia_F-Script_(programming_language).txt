<p><b>F-Script</b> is an <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> <a href="/wiki/Scripting_language" title="Scripting language">scripting</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a> for <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple's</a> <a href="/wiki/MacOS" title="MacOS">macOS</a> operating system developed by Philippe Mougin.  F-Script is an interactive language based on <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>, using macOS's native <a href="/wiki/Cocoa_(API)" title="Cocoa (API)">Cocoa</a> API.
</p>

<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=F-Script_(programming_language)&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>F-Script is based on a pure object paradigm: every entity manipulated within the language is an object. Its base syntax and concepts are identical to those of the language Smalltalk (the canonical example of an object-oriented language) with specific extensions to support <a href="/wiki/Array_programming" title="Array programming">array programming</a> as in the language <a href="/wiki/APL_(programming_language)" title="APL (programming language)">APL</a>.
</p><p>F-Script provides an <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreted</a>, <a href="/wiki/Interactivity" title="Interactivity">interactive</a> environment with support for workspaces, which provide a rich set of functions including <a href="/wiki/Object_persistence" class="mw-redirect" title="Object persistence">object persistence</a>, <a href="/wiki/Distributed_object" title="Distributed object">distributed objects</a>, <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> (GUI) framework, <a href="/wiki/Database" title="Database">database</a> access, among other things.
</p>
<h2><span class="mw-headline" id="Syntax">Syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=F-Script_(programming_language)&amp;action=edit&amp;section=2" title="Edit section: Syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Like Smalltalk, F-Script's syntax is very simple, without requiring specific notation for <a href="/wiki/Control_flow" title="Control flow">control</a> structures which are provided in a unified manner by the <code>message send</code> operation. Unlike Smalltalk, F-Script provides specific notational extensions to support the <code>Array</code> class, using curly brackets to describe literal arrays, which may contain any F-Script expressions. For example,
</p><p><code>{1+3, 'name', true}</code>
</p><p>is a valid array <a href="/wiki/Literal_(computer_programming)" title="Literal (computer programming)">literal</a>. The empty array is denoted by <code>{}</code>. Arrays of arrays are supported transparently, since any array is just another object.
</p>
<h2><span class="mw-headline" id="Message_sending">Message sending</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=F-Script_(programming_language)&amp;action=edit&amp;section=3" title="Edit section: Message sending">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Message expressions in F-Script are similar to those in Smalltalk: they specify which object is the receiver of the message, which operation is called by the message, and any argument objects needed by the operation. F-Script supports unary, binary, and keyword messages.
F-Script message semantics are extended to support array programming by recognizing that an array operation, such as adding to numerical vectors, must be viewed as generating a number of messages relating the elements of the vectors involved. Thus, if:
</p><p><code>A = {1, 2, 3}</code>
</p><p>and:
</p><p><code>B = {10, 20, 30}</code>,
</p><p>then F-Script allows:
</p><p><code>A + B = {11, 22, 33}</code>
</p>
<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=F-Script_(programming_language)&amp;action=edit&amp;section=4" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>F-Script is chiefly used as a lightweight scripting layer on top of <a href="/wiki/MacOS" title="MacOS">macOS</a>'s <a href="/wiki/Cocoa_(API)" title="Cocoa (API)">Cocoa</a> <a href="/wiki/Application_programming_interface" title="Application programming interface">application programming interface</a> (API). It can be embedded in applications using the F-Script framework and <a href="/wiki/Interface_Builder" title="Interface Builder">Interface Builder</a> palettes. It can also be used interactively from the F-Script interpreter to prototype applications. Finally, it can be used to explore applications' object hierarchies using an injector such as F-Script Anywhere.
</p>
